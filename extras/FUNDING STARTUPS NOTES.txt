The settings.py file
~~~~~~~~~~~~~~~~~~~~~
~~~~~~~~~~~~~~~~~~~~~
-> Why single settings.py is dangerous?

config/settings.py

Everything is inside ONE file:
debug
database
security
email
static files


Production risks:
debug: 	DEBUG = True (sensitive data leak)
database: Use MySQL or MS SQL Server or PostgreSQL
security: No HTTPS, weak secrets
secrets: API keys exposed in code


Industry Standard (solution)
----------------------------
config/
	settings/
		__init__.py
		base.py
		dev.py
		prod.py


Implementation
--------------
Step 1:

Creating 'settings' folder under config/

> mkdir config/settings

Create a file under 'settings' folder named __init__.py


Step 2:

Move settings.py -> base.py

config/settings.py	->  config/settings/base.py


Step 3:

Create dev.py and prod.py

config/settings/dev.py
config/settings/prod.py

----------------

Managing Secrets with .env (CRITICAL)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
- Stop hardcoding secrets in code
- Use .env file safely
- Load secrets into Django settings
- Prevent accidental leaks to GitHub


The problem (Very Dangerous)
-----------------------------

SECRET_KEY = 'django-insecure-e(w*8c30!g86-xh@z^iqwk_9^vb5efzg)cn_g=9uu#5e+@pwkp'

DATABASES = {
	'default': {
		'ENGINE': 'Django.db.backends.postgresql',
		'NAME': 'prod_db',
		'USER': 'postgres',
		'PASSWORD': 'secure_password',
		'HOST': 'localhost',
		'PORT': '5432'
	}
}



The Solution - .env file
-----------------------

Move the secrets into a separate file:

Environment:

SECRET_KEY = 'django-insecure-xyz123'
DEBUG = True
DB_NAME = mydb
DB_USER = postgres
DB_PASSWORD = supersecret


Install Required Package
------------------------
python-decouple

> pip install python-decouple


Create .env file
------------------
At project root (same level as manage.py)

SECRET_KEY = 'django-insecure-xyz123'

DB_USER = postgres
DB_PASSWORD = supersecret

DB_PORT = 5432


Update base.py
---------------
from decouple import config

# replace hardcoded values:
SECRET_KEY = config('SECRET_KEY')
DEBUG = config('DEBUG', default=False, cast=bool)
